## プロジェクト概要

## ソースファイル

- `SPARC.doc` — 元の Word ドキュメント
- `SPARC.pdf` — PDF 版（内容は同一）
- `GUIDELINE.md` — 作業方針

## 出力ファイル

- `SPARC.md` — 変換後の Markdown ファイル
- `images/` — 抽出した画像ファイルの格納先

## 画像について

文書には多数の図表が含まれる（約30点以上）。主な種類:

- **レジスタ構成図**: SPARCレジスタ一覧、%psr構造、%wim構造
- **スタック図**: 関数呼び出し時のスタック変化、スタック構造全体図
- **レジスタウインドウ図**: 円形のウインドウ図（関数呼び出し前後、オーバーフロー/アンダーフロー）
- **メモリ空間図**: ASIによるメモリ分離
- **キャッシュ構造図**: バンク/ライン構成
- **フロー図**: call/save/restore命令によるレジスタ値の変化、タスク切り替え手順、例外ハンドラのフロー

## 作業手順

### 第1段階（現在）: 画像抽出と Markdown 変換

1. PDF/DOC から図表を画像として抽出し `images/` に保存
2. `SPARC.md` を作成し、元文書と同じ内容を Markdown で記述
3. 画像は `![説明](images/ファイル名)` 形式で参照
4. テキスト部分は原文に忠実に変換

### 画像ファイル命名規則

`fig_セクション番号_連番_簡潔な説明.png`

例: `fig_01_01_sparc_registers.png`, `fig_03_01_save_window.png`

### 第2段階: 内容確認と修正

- 変換後の Markdown と元文書の突き合わせ確認
- 誤字脱字・レイアウト崩れの修正

### 第3段階: 公開

## 変換ルール

- 見出しは `#` 記法（セクション番号付き）
- コードブロック（アセンブリ命令等）は ``` で囲む
- 表は Markdown テーブル記法を使用
- 囲み枠のテキストは引用（`>`）またはテーブルで表現
- 箇条書きは `-` または `・` を使用
- 原文の内容・順序を変更しない
